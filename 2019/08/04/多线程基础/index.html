多线程基础

发表于 2019-08-04 更新于 2019-08-06 分类于 多线程

多线程的实现方式

实现Runnable接口

需要实现run()方法

通过Thread调用start()方法来启动线程

继承Thread类

实现run()方法，因为Thread类也实现了Runnable接口

调用start()方法启动线程，即该线程进入就绪状态，一旦被cpu调度则进入运行状态，运行run()方法

实现Callable接口

与Runnable相比，Callable可以有返回值，返回值通过FutureTask进行封装。

创建线程池

Executor 管理多个异步任务的执行，而无需程序员显式地管理线程的生命周期。这里的异步是指多个任务的执行互不干扰，不需要进行同步操作。

主要有三种Executor：

  • CachedThreadPool：一个任务创建一个线程
  • FixedThreadPool：所有任务只能使用固定大小的线程
  • SingleThreadPool：相当于大小为1的FixedThreadPool

与线程有关的常用方法

Thread中的方法

  • sleep()方法，让线程进入休眠
  • yield()方法，该线程让出cpu执行权，让线程之间重新竞争（可能这个线程会再次获得执行权）
  • join()方法，在线程中调用另一个线程的 join() 方法，会将当前线程挂起，而不是忙等待，直到目标线程结束。

Object中的方法

wait(),notify(),notifyAll()这几个方法必须在同步方法或者同步代码块中使用（与synchronized关键字配套使用），否则会在运行时抛出 IllegalMonitorStateException。

  • wait()方法，调用 wait() 使得线程等待某个条件满足，线程在等待时会被挂起，当其他线程的运行使得这个条件满足时，其它线程会调用 notify() 或者 notifyAll() 来唤醒挂起的线程。
  • notify()方法，随机唤醒一条正在等待的线程
  • notifyAll()方法，唤醒全部正在等待的线程

区别

  • sleep方法没有释放锁，wait方法释放了锁
  • wait，notify和notifyAll方法都只能在同步方法或同步代码块中使用，而sleep方法没有此限制
  • sleep方法必须捕获异常，而wait，notify与notifyAll不需要捕获异常

注意：

（1）sleep

  • sleep方法表示让一个线程进入睡眠状态，等待一段时间后，自动醒来进入到可运行状态，不会马上进入运行状态，因为线程调度机制恢复线程的运行也需要时间。
  • 但在sleep的过程中过程中有可能被其他对象调用它的interrupt(),产生InterruptedException异常，如果你的程序不捕获这个异常，线程就会异常终止，进入TERMINATED状态，如果你的程序捕获了这个异常，那么程序就会继续执行catch语句块(可能还有finally语句块)以及以后的代码。

（2）wait

  • wait属于Object的成员方法，一旦一个对象调用了wait方法，必须要采用notify()和notifyAll()方法唤醒该进程
  • 如果线程拥有某个或某些对象的同步锁，那么在调用了wait()后，这个线程就会释放它持有的所有同步资源，而不限于这个被调用了wait()方法的对象。wait()方法也同样会在wait的过程中有可能被其他对象调用interrupt()方法而产生

线程的状态

  • 新建（New），未启动
  • 可运行状态（Runnable），start()
  • 运行状态（Running），run()
  • 阻塞状态（Blocked），sleep()
  • 等待状态（Waiting），wait()
  • 销毁状态（Terminated），run()执行结束或者出现异常停止
